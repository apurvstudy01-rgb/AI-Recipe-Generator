# 🍛 AI Recipe Generator

An AI-powered web application that generates complete Indian recipes from just a few ingredients. Built with a **Hybrid Retrieval-Augmented Generation (RAG)** architecture combining a smart ingredient matching algorithm with a trained Bidirectional LSTM neural network.

> **Live Demo:** [Click here to try it](https://huggingface.co/spaces/Apurv07/AI-Recipe-Generator)
> 
> **GitHub Repo:** (github.com/YOUR_USERNAME/AI-Recipe-Generator)

## 🧠 How It Works

```
User types: "paneer, spinach"
            ↓
Weighted Scoring Algorithm searches all 6,870 recipes
  +3 points → ingredient found in recipe NAME
  +2 points → ingredient found in ingredients LIST
  +0.5 bonus → each additional ingredient matched
            ↓
Best match found: "Palak Paneer Recipe" ✅
            ↓
Shows REAL recipe from dataset:
  name, cuisine, course, diet,
  prep time, cook time, servings,
  full ingredients list, cooking instructions
            ↓
LSTM Neural Network generates AI Chef's Tip 🤖
```

This architecture is called **Retrieval-Augmented Generation (RAG)** — the same concept used by ChatGPT with web search.

## 🤖 LSTM Model Architecture

```
Embedding Layer (vocab_size × 100)
        ↓
Bidirectional LSTM (150 units, return_sequences=True)
        ↓
Dropout (0.2)
        ↓
LSTM (100 units)
        ↓
Dense (vocab_size/2, ReLU)
        ↓
Dense Output (vocab_size, Softmax)
```

**Training Details:**
- Dataset: 6,870 Indian recipes (1,000 used for CPU training)
- Loss: Sparse Categorical Crossentropy
- Optimizer: Adam
- Epochs: 20 | Batch Size: 256
- Sequence Length: 50 tokens (capped)
- Role: AI Chef's Tip generation only

## 🚀 Local Setup (Windows)

### Step 1 — Clone the Repository
```cmd
git clone https://github.com/YOUR_USERNAME/AI-Recipe-Generator.git
cd AI-Recipe-Generator
```

### Step 2 — Create Virtual Environment
```cmd
python -m venv venv
venv\Scripts\activate
```

### Step 3 — Install Dependencies
```cmd
pip install -r requirements.txt
```

### Step 4 — Train the Model (One-time, ~20 minutes on CPU)
```cmd
python train_model.py
```

This creates three files:
- `recipe_generation_model.keras`
- `tokenizer.pkl`
- `model_config.pkl`

### Step 5 — Run the App
```cmd
python app.py
```

Open your browser at: **http://127.0.0.1:7860**

## ☁️ Deployment (Hugging Face Spaces)

This project is deployed on **Hugging Face Spaces** (free, 16 GB RAM, Docker support).

**Why Hugging Face and not Render.com?**
- Render free tier = 512 MB RAM → TensorFlow alone crashes it
- Hugging Face free tier = 16 GB RAM → works perfectly for ML models

**Model files** are stored on Google Drive and downloaded automatically at startup via `download_model.py` — so no training happens on the server.

### Deploy Your Own
1. Fork this repository on GitHub
2. Create a new Space on [huggingface.co/spaces](https://huggingface.co/spaces) → SDK: Docker
3. Upload all project files to the Space
4. Update `download_model.py` with your own Google Drive file IDs
5. The Space builds automatically using the `Dockerfile`

## 🔧 Key Technical Fixes Applied

During development, several critical issues were identified and resolved:

| Problem | Fix Applied |
|---|---|
| `requirements.txt` had 200+ macOS-only packages | Cleaned to 5 essential packages |
| `fit_generator()` removed in TensorFlow 2.x | Replaced with `model.fit()` |
| `Dense(total_words/2)` TypeError in Python 3 | Changed to `Dense(int(total_words/2))` |
| `to_categorical()` needed 256 GB RAM | Switched to `sparse_categorical_crossentropy` |
| CPU training was taking 500+ hours | Limited recipes, shorter sequences, larger batch |
| LSTM generating wrong output (eggs for paneer) | Adopted hybrid RAG — dataset retrieval + LSTM tip |
| Wrong recipe matching (Aloo Paratha for paneer) | Built weighted scoring: +3 name, +2 ingredients, +0.5 bonus |
| Render.com crashed (512 MB RAM limit) | Migrated to Hugging Face Spaces (16 GB RAM) |
| Server trying to train model on deployment | Model stored on Google Drive, downloaded at startup |
| Missing HTML templates in original repo | Built complete UI from scratch with futuristic dark theme |
| Tokenizer not saved — word index mismatch | Tokenizer saved as `tokenizer.pkl` during training |
